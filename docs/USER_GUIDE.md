# 筑迹 使用手册

> 写给业主的图文说明。所有功能都讲到，不需要技术背景。

筑迹是一个**装修管家**：帮你盯紧节点进度、记录每一笔采购、生成档案。所有数据保存在你**自己浏览器本地**，不上传任何服务器。

---

## 1. 第一次打开

打开网页后，会看到欢迎卡片，有两个按钮：

- **新建项目** — 创建一个空白的装修项目（输入项目名、面积、预算等）
- **加载示例项目** — 一键加载一个 89㎡ 演示项目（约 30 笔采购），直接体验全部功能

**建议第一次先点「加载示例项目」**，把所有功能玩一遍，再回头新建自己的真实项目。

---

## 2. 主界面四个视图

顶部导航有四个按钮：

| 视图 | 用途 |
| :--- | :--- |
| **总览** | Dashboard：进度环、累计支出、本周采购、阶段花费、品类饼图、Top 5 高价采购、采购趋势 |
| **节点工作台** | 11 个阶段、~60 个节点的进度管理，每个节点有避坑要点 / Checklist / 采购 / 图片 / 备注 5 个 Tab |
| **采购流水** | 全部采购的清单视图，支持按阶段 / 节点 / 品类筛选、关键字搜索、Excel 导出 |
| **时间线** | 自研甘特图，按阶段分组展示节点条，**可以直接拖拽节点条调整计划日期** |

右上角还有：

- **+ 记一笔** — 快速打开记账抽屉（快捷键 `⌘N`）
- **🔔 提醒** — 查看 / 添加提醒
- **⚙️ 设置** — 项目设置 / 模板管理 / 备份 / PDF 导出

---

## 3. 节点工作台 — 装修过程的主战场

### 3.1 阶段 → 节点

左侧是阶段树（"前期准备 / 拆改 / 水电改造 / 防水 / 瓦工…"共 11 个阶段）。每个阶段下面有若干**节点**（如"水路改造"、"电路改造"）。

点开一个节点，右侧会出现 5 个 Tab：

| Tab | 内容 |
| :--- | :--- |
| **避坑清单** | 这个节点常见的坑（如"水管走顶不走地"、"开槽深度不超过 1/3 墙厚"） |
| **Checklist** | 这个节点要做的事，勾选后实时更新进度 |
| **采购** | 挂在这个节点上的所有采购明细，可以从这里直接「+ 记一笔」 |
| **图片** | 节点照片 + 该节点下采购的图片，**点缩略图打开灯箱**（键盘 ←/→/+/-/0/Esc，手机单指滑动 / 双指捏合 / 双击放大） |
| **备注** | 富文本备注（可加粗、列表、链接） |

### 3.2 节点状态

每个节点有 4 个状态：

- ⚪ 未开始
- 🟡 进行中
- 🟢 已完成
- ⚫ 已跳过（这一项不做）

点节点头部的按钮切换。**已完成**的节点会贡献到总览页的"进度环"。

### 3.3 上传图片

节点的「图片」Tab 里点「上传」选图就行。

- 自动压缩到长边 ≤1920px、JPEG 85% 质量（PNG 保留透明），照片清晰但不占空间
- 一次可以选多张
- 图片缩略图懒加载，不卡

## 4. 记一笔 — 怎么记录采购

任何位置（除了灯箱）按 `⌘N` 或点右上「+ 记一笔」都能打开记账抽屉。

字段说明：

| 字段 | 例子 | 说明 |
| :--- | :--- | :--- |
| 名称 | 马可波罗通体大理石瓷砖 | **必填** |
| 规格 | 800×800 | 可选 |
| 品牌 | 马可波罗 | 可选 |
| 渠道 | 天猫旗舰店 | 可选 |
| 单价 × 数量 | 89 × 30 = ¥2,670 | **单价和数量都必须 > 0** |
| 购买日期 | 2026-04-15 | **必填** |
| 购买链接 | https://detail.tmall.com/... | 可选，会渲染成可点击的链接 |
| 图片 | （上传） | 可选 |
| 挂到节点 | 瓦工 / 客厅铺砖 | 可选，不挂也行 |
| 品类 | 主材 / 辅材 / 家具 / 软装 / 家电 / 人工 / 其他 | 影响品类饼图 |
| 备注 | 含送货上楼 | 可选 |

校验：单价 / 数量必须 > 0，日期必填。失败会有 toast 提示。

---

## 5. 采购流水 — 找钱花到哪里去了

视图顶部的工具栏有：

- 阶段下拉 / 节点下拉 / 品类下拉（三个独立筛选，叠加生效）
- 关键字搜索（搜名称 / 品牌 / 规格 / 备注）
- 「导出 Excel」按钮（导出当前筛选结果，**xlsx 是按需加载的，第一次点会稍慢**）

点击任意行打开该笔采购的编辑抽屉，可以改 / 删。

**总览页里点「阶段花费分布」条会自动跳转到这里 + 预选该阶段**——这是排查"这个阶段花了多少"的最快方式。

---

## 6. 时间线 — 看节点排期

按阶段分组显示节点条，左侧是节点名，右侧是时间轴。

**可以直接拖拽节点条**：

- 拖中间 = 整体平移（开始 + 结束一起移）
- 拖左边缘 = 调整开始日期
- 拖右边缘 = 调整结束日期

拖完自动保存。

如果一个节点还没设置日期，会显示在底部的"未排期"区，点编辑可以填日期。

---

## 7. 全局搜索 — `⌘K`

任何时候按 `⌘K`（Mac）或 `Ctrl+K`（Windows）打开命令面板：

- 输入关键字，模糊匹配节点 / 采购 / 提醒
- 上下方向键选择，**回车跳转**
- Esc 关闭

---

## 8. 提醒

点右上「🔔 提醒」打开提醒中心。

- 三种来源：**节点提醒**（节点开始 / 结束前 N 天）、**采购提醒**（如"3 个月后看砖面是否泛碱"）、**自定义提醒**（任意时间任意内容）
- 双通道：到点会**弹应用内 toast** + **触发系统 Notification**（首次会弹权限请求；浏览器不允许就只有 toast）
- 已触发的提醒会标记为已读，可以重置

## 9. 数据备份（**重要！**）

筑迹的数据完全在你浏览器本地（IndexedDB）。**清空浏览器数据 / 卸载浏览器 / 换电脑前，一定先备份**。

设置 → 数据 区域有 3 种备份方式：

### 9.1 完整 Zip 备份（**最稳妥**，建议每周一次）

点「导出备份 Zip」。下载一个 `筑迹-备份-YYYY-MM-DD.zip`，里面有：

- `manifest.json` — 全部项目 / 节点 / 采购 / 提醒数据
- `assets/` — 所有上传的原图

换电脑 / 重装浏览器后，进设置点「导入备份 Zip」选这个文件，一键恢复。

### 9.2 本地目录镜像（Chrome / Edge 限定）

点「选择镜像目录」，授权一个本地文件夹。从此：

- 每次数据变更，2 秒内同步写入 `筑迹/projects/<projectId>/data.json` + `images/`
- 每天自动生成一个完整 Zip 到 `筑迹/snapshots/<日期>.zip`，保留最近 30 天

**把这个目录放在 iCloud / OneDrive / Dropbox 里**，自动多端同步、自动版本历史，强烈推荐。

Safari / 火狐不支持 File System Access API，会降级为"下载快照 Zip"按钮，需要手动下载。

### 9.3 PDF 装修档案

点「生成 PDF」，浏览器会新开一个窗口展示可打印的档案：

- 项目信息（名称 / 面积 / 预算 / 起止日期）
- 节点进度（11 个阶段、~60 个节点的状态）
- 阶段花费汇总
- 采购流水明细

按 `⌘P` / `Ctrl+P` 打印为 PDF 即可归档。**适合装修结束后留个纸质 / 电子档案**。

## 10. 模板管理 — 改不喜欢的预置内容

筑迹自带 11 个阶段、~60 个节点、每个节点的避坑要点 + Checklist。如果你觉得"水电改造里漏了开关插座定位"或者"这个 Checklist 项不适用"，可以改：

设置 → 节点模板管理 →「编辑模板」

- 增删阶段 / 节点
- 编辑每个节点的预置避坑要点 / Checklist 项
- 改完点「保存」

**注意：模板变更只影响新建的项目**，已经存在的项目不会被改写（防止你已经勾的 Checklist 被覆盖）。

如果改坏了，点「恢复出厂模板」一键还原。

---

## 11. 项目切换

顶部项目名右边有「切换项目」按钮：

- 点开下拉菜单，可以新建项目 / 切换项目 / 编辑当前项目 / 删除当前项目
- 每个项目的数据完全隔离（Dashboard / 节点 / 采购都按项目过滤）
- 删除项目要二次确认，删除后**数据不可恢复**（除非你之前导过 Zip）

---

## 12. PWA — 安装到桌面

筑迹是 PWA（Progressive Web App），可以像 App 一样装到桌面：

- **Chrome / Edge**：地址栏右侧会出现一个"安装"图标，点一下确认即可
- **Safari (iOS)**：分享菜单 → 添加到主屏幕
- **Safari (macOS)**：菜单栏 文件 → 添加到 Dock

装完之后：

- 桌面 / 启动台有一个独立图标
- **离线可用**（首次加载后，Service Worker 会缓存全部资源；没网也能打开看数据 / 记账）
- 全屏运行，没有浏览器地址栏

## 14. 常见问题

**Q: 数据真的不上传服务器吗？**
A: 真的。所有数据都在浏览器 IndexedDB 里。打开浏览器 DevTools → Application → IndexedDB → `zhuji-db` 可以亲眼看见。

**Q: 多个设备怎么同步？**
A: 用「本地目录镜像」+ iCloud / OneDrive / Dropbox（见 9.2）。换设备时，把镜像目录里的最新 Zip 在新设备上导入即可。

**Q: 同一个浏览器开多个标签会冲突吗？**
A: 不会，IndexedDB 是线程安全的。但建议不要同时在多个标签上**编辑同一笔采购**。

**Q: 升级后我的数据会丢吗？**
A: 不会。Dexie schema 升级是兼容的。但凡是大版本升级前，**先导一份 Zip 备份**是个好习惯。

**Q: 我能自定义阶段吗？**
A: 能（见 11 模板管理）。但只影响新建项目。

**Q: 微信 / 钉钉里打开的浏览器能用吗？**
A: 能用，但浏览器内核较老，**本地目录镜像和 PWA 安装功能用不了**。建议外部浏览器（Chrome / Edge / Safari）打开。

---

## 15. 出问题了？

筑迹有全局 ErrorBoundary，遇到 JS 错误会显示一个"出错了 + 重试"的界面，不会白屏。

如果某个功能完全不工作：

1. **导一份 Zip 备份**保命
2. 打开 DevTools (F12) → Console 看红色错误
3. 试试硬刷新（`⌘Shift+R` / `Ctrl+Shift+R`）
4. 实在不行清空浏览器数据后导回 Zip
